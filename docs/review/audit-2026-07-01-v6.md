### 2.2 P2 — 计划修复

#### ITM-048 · `[GenerateMessage]` 属性的 Name/SchemaVersion 是 public setter [可信度 ✅]

- **维度**：可维护性 / DDD 不变量
- **危害**：中 — 调用方可在运行时修改 Name/SchemaVersion，破坏 wire name 稳定性
- **证据**：`src/PalDDD.Core/Attributes.cs:40,43`
  ```csharp
  public string? Name { get; set; }           // 行40 — 可被外部修改
  public int SchemaVersion { get; set; } = 1; // 行43 — 可被外部修改
  ```
- **论证**：`[GenerateMessage(Name = "...", SchemaVersion = 1)]` 是特性声明，正常使用不会修改。但 `public set` 允许运行时反射修改（虽然框架禁反射，但外部消费方可能用反射）。DDD 原则要求不变量封装——wire name 应该是不可变的。
- **权衡**：改为 `init` 会破坏 .NET 特性初始化语法（特性只支持 `set`，不支持 `init`）。因此**不可修复**——这是 .NET Attribute 的语言限制。建议在 XML doc 中标注"Name/SchemaVersion 应只在声明时设置，运行时修改会导致 wire name 漂移"。
- **修复**：文档级。在 `GenerateMessageAttribute` XML doc 增加 `<remarks>` 警告。

### 2.3 P3 — 评估

#### ITM-049 · IPalOutboxStore 定义在 Transactions 层而非 Core 层 [可信度 ✅]

- **维度**：Clean Architecture 依赖反转
- **证据**：`IPalOutboxStore` 定义在 `src/PalDDD.Transactions/OutboxStore.cs:13`（App-Core 层），`IUnitOfWork` 定义在 `src/PalDDD.Core/IUnitOfWork.cs:16`（Domain 层）
- **论证**：`IUnitOfWork` 在 Core 层是因为它是最基本的事务抽象。`IPalOutboxStore` 在 Transactions 层——这本身是合理的（Outbox 是 Transactions 的概念）。但消费方需要引用 `PalDDD.Transactions` 才能使用 `IPalOutboxStore`，这增加了耦合。
- **判定**：设计合理——Outbox 不是领域概念，属于应用层。**不需要修复**。

#### ITM-050 · HandlerNotFoundException 错误消息已包含修复建议 ✅ [可信度 ✅]

- **维度**：消费者视角 · 错误友好度
- **证据**：`HandlerNotFoundException.cs:41` — `"未找到请求类型 '{requestType.Name}' 的处理器。请使用 AddPalCommandHandler / AddPalQueryHandler 显式注册处理器。"`
- **判定**：**优秀**——错误消息包含类型名 + 修复方法。无需改进。

## 第三部分：综合评估

### 3.1 DDD 建模质量

| 检查项 | 结果 | 证据 |
|--------|:--:|------|
| 聚合根是薄基类 | ✅ | `AggregateRoot<TId>` 仅继承 `Entity<TId>`，无额外逻辑 |
| Entity 使用单链表存储事件 | ✅ | O(1) 追加，零容器分配，ref struct 枚举器 |
| 聚合根无 public setter | ✅ | 仅 Attributes 的 Name/SchemaVersion（.NET 限制，ITM-048） |
| 值对象用 readonly record struct | ✅ | `ValueObject<T>` + `ExpectedStreamVersion` |
| 值对象工厂方法含验证 | ✅ | `ExpectedStreamVersion.Exact()` 含 `ThrowIfLessThan(0)` |
| BC 边界由 PDDD001 强制 | ✅ | 所有领域模型类型必须声明 `[BoundedContext]` |
| 事件版本化由 PALMSG001-005 强制 | ✅ | Name 非空 + SchemaVersion ≥ 1 |
| DIM 桥接替代反射 | ✅ | `IHandler.HandleAsync` 默认接口方法 |
| 跨 BC 仅领域事件 | ✅ | 无 EventBus |
