# ADR-016：IValueObject 校验契约评估

> 状态：已采纳 · 日期：2026-07-01 · 关联：ITM-059

## 背景

`IValueObject` 接口为标记接口，不包含任何方法。`ValueObject<T>` 构造函数不做验证，依赖派生类型自行实现。已通过 ADR-003 评估`IValueObject` 保留。

## 决策

**不增加 `Validate()` 或 `IsValid()` 契约方法。**

## 理由

1. **派生类型已在构造函数中校验**：值对象的不可变性决定了校验时机在构造时——`ExpectedStreamVersion.Exact(version)` 工厂方法包含 `ThrowIfLessThan(0)`，非数值类型在 `Create` 工厂方法中校验。
2. **基类验证会引入不必要的耦合**：不同类型的值对象有不同的校验需求——数值需要范围检查、字符串需要格式检查、GUID 不需要检查。统一的 `IsValid()` 接口无法表达不同校验语义。
3. **YAGNI**：当前所有值对象实现都在构造/工厂方法中完成了校验，无消费方需要"运行时检查值对象是否有效"的功能。增加契约方法不会解决任何实际存在的问题。
4. **基类构造函数接受 `Validator<T>` 委托违反简洁性**：`readonly record struct` 的简洁性是其核心价值，增加委托参数会破坏零分配路径。

## 不动

维持现状。值对象的校验责任由派生类型的构造/工厂方法承担。
