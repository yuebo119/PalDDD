# ADR 003：IValueObject 标记接口保留决策

> 状态：已采纳  
> 日期：2026-06-28  

## 背景

`IValueObject` 是一个空的标记接口（9 行代码），当前代码中没有 `if (x is IValueObject)` 模式匹配。按照"无引用即死代码"的简化判断，可能被误判为可删除。

## 决策

**保留 `IValueObject`，不删除。**

## 理由

1. **DDD 语义价值**：`IValueObject` 是 Ubiquitous Language 的一部分。它告诉框架使用者"这个类型是值对象"——在 IDE 类型层次、文档生成、EF Core 配置约定检测中都有实际价值。

2. **源码生成器契约**：未来可基于此接口扩展源码生成器，如自动生成值对象的 EF Core 值转换器、JSON 转换器、相等性比较器。

3. **框架扩展性**：第三方扩展可以通过 `where T : IValueObject` 约束提供通用值对象服务（批量验证、批量序列化注册）。

4. **维护成本远低于语义价值**：9 行接口代码的维护成本极低，但它提供的 DDD 语义标记能力在长期框架演化中不可替代。

## 后果

- 保留 9 行接口代码
- 源码注释精简为一行 why + 指向本 ADR 的链接
- 未来如出现基于此接口的源码生成器扩展，无需重新引入
